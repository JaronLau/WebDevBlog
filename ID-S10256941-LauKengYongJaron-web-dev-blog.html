    Forms - Positioning - Media Queries - Flex

    Forms

    A HTML form is to collect user input and send it to a server for processing.
    The form element acts as a container for the input elements to collect different user inputs.
    Typees of input elements include
    • text
    • radio
    • checkbox
    • submit
    • button
    Labels are to show what the forms are for
    Label for 'fname' = First name:,
    Input type='text' id='fname' name='fname'



    Positioning

    Positioning property changes the positioning scheme of the element

    Types of positioning include:
    • Relative position
      • moves element relative to where it would be on the page
    • Absolute position
      • Elements are fixed in relation to the parent content
      • The offset is relative to the browser window
    • Fixed position
      • Element coordinate is always relativce to the entire browser window
      • elements do not scroll with page; sticky element
    • z-index
      • Controlls the depth of the element on the page
      • Elements with higher z-index overlapps those with lower value

    Media Queries

    The @media rule allowed for different styles according to different media types

    E.g. Phone screens can have different layouts and color schemes compared to the website on a desktop.
    • @media screen and (min-width:400px){styles to be added here e.g. body{bg:green}}

    Flex

    Flexbox container is the parent of the flex element;children.

    Children can be positioned and and flexible according to code and user needs
  • children can be positioned horizontally and vertically
  • children can grow and shrink according to window size

    Object, Arrays, and Manipulation of Arrays

    Objects

    Objects are variables that contain properties.
    • E.g. var person = {name:"Joe", age:"30", eyeC:"brown"}
    • We can access the property values by using "person.eyeC" to return property: brown

    Arrays

    Arrays are like python lists, adding items in arrays to store for easy loops.

    Arrays preserve order of items placed in them.
    • E.g. bird = ["crow", "raven", "magpipe"]
    • bird[0] returns "crow"
    • bird[0]="quail" replaces crow with quail in the array
    • Adding elements using bird[bird.length]="eagle"
    • Removing elements using bird.splice(2,1) removes "magpipe"

    Comments and whitespace

    • Comments are written using //comments after the backslashe
    • comment blocks span across multiple lines until it is closed using /*this comment spans*/
    • Whitespace only affects readability of code, JS mostly ignores whitespace

    Iterating with Loops

    Loops in JS cover for loops, enumerative loops, while loops.

    For loops loop though the code for a set number of times

    While loops loops through the code until a condition is fufilled

    Enumerative loops loops over object or arrays
    • const person = {fname:"John", lname:"Doe", age:25};
      let text = "";
      for (let x in person) {
      text += person[x];}
      produces the text 'John Doe 25'
    • const numbers = [45, 4, 9, 16, 25];
      let txt = "";
      for (let x in numbers) {
      txt += numbers[x];}
      produces 45 4 9 16 25
